# Голосовые комментарии

## ОБЯЗАТЕЛЬНО: голос

Перед **каждым** вызовом `mcp__tts__tts` выполни:
```bash
echo $TTS_VOICE
```
и передай результат в параметр `voice`. Никогда не используй дефолтный голос из схемы.

---

Используй инструмент `mcp__tts__tts` для озвучивания в трёх случаях:

## 1. Перед началом работы — короткий анонс (1 предложение)
Что делаешь прямо сейчас. Только если предстоит нетривиальная работа.

## 2. По ходу работы — важные находки
Только если обнаружил что-то значимое: баг, неожиданное поведение, проблему.

## 3. Итоговый ответ
Финальное сообщение пользователю — всегда озвучивай с `summary=true`.

## Не озвучивай
- Код, bash-команды, пути к файлам
- Технические выводы инструментов
- Английский текст
- Промежуточные шаги без смысловой ценности

## Параметры вызова

| Параметр    | Тип     | Описание |
|-------------|---------|----------|
| `voice`     | string  | результат `echo $TTS_VOICE` — всегда |
| `rate`      | float   | скорость речи: 0.5–2.0, норма = 1.0 |
| `pitch`     | float   | высота тона: 0.0–1.5, норма = 1.0 |
| `normalize` | boolean | `true` всегда |
| `summary`   | boolean | `false` для коротких фраз, `true` для итогового ответа |

**Управление настроением через rate + pitch:**

| Настроение | rate | pitch |
|------------|------|-------|
| нейтрально | 1.0  | 1.0   |
| спокойно   | 0.8  | 0.8   |
| деловито   | 1.1  | 1.0   |
| радостно   | 1.2  | 1.2   |
| серьёзно   | 0.9  | 0.85  |
| драматично | 0.5  | 0.7   |
| срочно     | 1.4  | 1.1   |

## Управление произношением в тексте

При rate=1.0 и pitch=1.0 Silero использует авто-ударение. При отклонении — SSML prosody.
Можно явно управлять прямо в тексте:

**1. Ударение через `+` перед ударной гласной** (для омографов):
```
Зам+ок (замок-строение) vs з+амок (замок-механизм)
Стр+елки (стрелки часов) vs стрелк+и (люди)
```

**2. Паузы через точки или пробелы:**
```
Подождите.. немного — пауза через точки
```

**3. SSML `<break>` для точного контроля паузы:**
```xml
<speak>Подождите <break time="500ms"/> продолжаем</speak>
```
